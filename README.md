# 车载 U 盘音乐整理与格式转换实践

## 项目简介

本项目记录了一次将本地音乐文件整理为车载 U 盘音乐库的实践过程。实践车辆为 2016 年购买的尼桑轩逸，目标是解决车机对音乐文件格式、U 盘格式、目录结构和文件命名的兼容性问题，最终形成一套稳定、可复用的车载音乐整理方案。

本实践重点围绕以下问题展开：

- MP3 文件应该放在 U 盘根目录，还是新建文件夹；
- 2016 年左右的尼桑轩逸是否适合使用 FLAC；
- FLAC、MP3、NCM 三种格式的区别；
- 如何将 FLAC 转换为 MP3；
- PowerShell 中 `ffmpeg` 无法识别时如何处理；
- 车载系统是否支持歌词显示；
- 如何沉淀成可复用的实践文档和脚本。

最终结论：对于 2016 年左右的轩逸车机，推荐使用 **FAT32 格式 U 盘 + 320kbps MP3 + 简单目录结构**。

## 实践背景

在将音乐导入 U 盘并用于车载播放时，不能只考虑音质，还需要考虑车机兼容性。尤其是 2016 年左右的老款车机，对文件系统、音频格式和目录层级的支持通常比较保守。

本次实践发现，车载音乐整理的核心目标不是追求最高音质，而是保证：

1. 车机能够识别 U 盘；
2. 车机能够扫描到音乐文件；
3. 音乐能够稳定播放；
4. 歌曲排序和文件管理足够清晰；
5. 后续新增歌曲时有统一规范。

---

## 推荐方案

### U 盘格式

推荐使用：

```text
FAT32
```

原因是 FAT32 对老款车机兼容性较好。虽然 exFAT 和 NTFS 在电脑上更常见，但部分老车机可能无法识别。

### U 盘容量

推荐优先选择：

```text
16GB 或 32GB
```

对于车载 MP3 音乐来说，这个容量已经足够使用。容量过大的 U 盘可能在老车机上出现识别不稳定的问题。

### 音乐格式

推荐使用：

```text
MP3
320kbps
44.1kHz
双声道
```

对于车载环境，320kbps MP3 在音质、文件大小和兼容性之间比较均衡。

## 文件夹结构建议

### 方案一：直接放在根目录

```text
U盘
├── 001 晴天.mp3
├── 002 夜曲.mp3
├── 003 稻香.mp3
└── 004 江南.mp3
```

这是兼容性最强的方式，适合老车机。

### 方案二：建立一级 MUSIC 文件夹

```text
U盘
└── MUSIC
    ├── 001 晴天.mp3
    ├── 002 夜曲.mp3
    ├── 003 稻香.mp3
    └── 004 江南.mp3
```

这种方式更整洁，也比较适合车载 U 盘。

### 不推荐的结构

```text
U盘
└── 音乐
    └── 华语
        └── 歌手
            └── 专辑
                └── 歌曲.mp3
```

目录层级过深，老车机可能无法完整扫描。

## FLAC、MP3、NCM 的区别

| 格式 | 类型 | 特点 | 是否推荐用于车载 |
|---|---|---|---|
| FLAC | 无损音频格式 | 音质高，文件大 | 不推荐作为老车机主力格式 |
| MP3 | 有损音频格式 | 兼容性强，文件较小 | 推荐 |
| NCM | 网易云音乐加密封装格式 | 普通车机无法直接播放 | 不推荐，需要先转换 |

简单理解：

```text
FLAC = 适合收藏的无损音乐格式
MP3 = 适合车载播放的通用音乐格式
NCM = 网易云音乐的加密封装文件
```

对于本次实践车辆，最适合的方案是：

```text
电脑收藏：FLAC
车载使用：320kbps MP3
网易云下载文件：NCM 需要先转换
```

## FLAC 批量转换为 MP3

本项目提供了一个 PowerShell 脚本：

```text
scripts/flac_to_mp3.ps1
```

该脚本会将当前目录下的 `.flac` 文件批量转换为 320kbps MP3，并输出到 `mp3` 文件夹中。

### 使用前准备

需要先安装 FFmpeg。

在 PowerShell 中执行：

```powershell
winget install -e --id Gyan.FFmpeg
```

安装完成后，关闭当前 PowerShell，重新打开，再输入：

```powershell
ffmpeg -version
```

如果能看到版本信息，说明 FFmpeg 安装成功。

### 使用方法

将 `flac_to_mp3.ps1` 复制到 FLAC 音乐所在文件夹，然后在该文件夹中打开 PowerShell，执行：

```powershell
.\flac_to_mp3.ps1
```

转换完成后，MP3 文件会输出到：

```text
当前文件夹\mp3
```

## FFmpeg 无法识别的处理

如果 PowerShell 提示：

```text
ffmpeg : 无法将“ffmpeg”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。
```

说明系统无法找到 FFmpeg。常见原因包括：

1. FFmpeg 没有安装；
2. FFmpeg 没有加入系统环境变量 PATH；
3. 安装后没有重新打开 PowerShell。

解决方式：

```powershell
winget install -e --id Gyan.FFmpeg
```

然后重新打开 PowerShell，执行：

```powershell
ffmpeg -version
```

## 歌词显示说明

对于 2016 年左右的轩逸原车车机，大概率不能显示滚动歌词。即使将 `.lrc` 歌词文件与 MP3 放在同一目录下，车机也不一定能识别。

车机通常更可能显示以下信息：

```text
歌曲名
歌手
专辑
播放时间
曲目编号
```

因此，车载音乐整理的重点应放在 MP3 文件本身、文件名和标签信息，而不是歌词显示。

## 实践结论

本次实践形成的核心经验是：

> 车载音乐整理应优先考虑稳定性和兼容性，而不是单纯追求最高音质。

对于 2016 年左右的尼桑轩逸，推荐最终方案如下：

```text
U盘格式：FAT32
U盘容量：16GB 或 32GB
音乐格式：MP3
码率：320kbps
采样率：44.1kHz
声道：双声道
目录结构：根目录或 MUSIC 一级文件夹
文件命名：001 歌名.mp3
```

该方案可以较好地解决老车机常见的识别失败、播放失败、排序混乱等问题。
